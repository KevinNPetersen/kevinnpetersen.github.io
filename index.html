XBOX LIVE DOS ATTACK

What do you do when your roommate chooses to play Halo online instead of doing his dishes? You hack away!

All I need is Kali Linux, a wireless adapter and a command prompt for the fun to begin.

Booting into Kali Linix and opening a command prompt I'll verify my wireless adapter is set to monitor mode. Running the command:iwconfig displays the adapter settings which are correct.

Next we want to scan the area to find our network. Running the command airodump-ng will do the trick. Just make sure to add your wireless adapter at the end.

I'll look for my networks name and copy the BSSID and take note of the Channel (CH) number.

Next we'll run the following:
  • Our program:airodump-ng
  • Our networks BSSID:--bssid XX:XX:XX:XX:XX:XX
  • The channel:--channel X
  • Our adapter:wlan0

    • The results gives me a list of devices and with a little trial and error I'm able to obtain my roommates Xbox MAC address. The final key to the puzzle!

    At last we'll run the final command. Lets break down what we need:
  • Main program:aireplay-ng
  • Attack command with frame count:-0 xxx
  • Access point MAC:-a xx:xx:xx:xx:xx:xx

    Rogue Access Point

    PLEASE NOTE!
    This was done for experimental purposes only and was never used outside a controlled environment.

    For this experiment I'll be using a WEMOS D1 Mini V4.0.0 IoT WiFi board that can be bought for as low as $1.60. I'll be building a captive portal that will mimic a nearby Starbucks, utilizing the blueprint provided by the code supplied here. I'll be at a slight disadvantage thanks to my device not being connected to any network or storage, so I'll have to create the interface without the help of icons and pictures.

    I'll be using Arduino IDE to compile the code and transfer it to the device.

    Lets start with tricking the user into clicking on my WiFi by renaming my device to "Starbucks WiFi". The fake WiFi network is listed at the top, and the real Starbucks WiFi network is listed at the bottom.

    Once connected the device will bring up a captive portal for first time connectors. Now sadly this is where I'm at a disadvantage and will require the user to be less suspicious and in dire need of WiFi. Even though mine on the left lacks the aesthetics, pictures and charm as the legitimate network reflected on the right, someone less technically inclined could fall victim and submit their information without a second thought.

    Once a user types their information and presses "Accept", they'll be brought to a Validated screen which ultimately leads no where.

    From there I've just stolen your credentials! An email and zip code might not compromise much, but a more sophisticated attack could gather passwords or account numbers.

    This project exposes how dangerous a free public WiFi network can be. Whether its at a restaurant, mall or airport, your personal data can be compromised. All it takes is a $2 device and skilled hacker with malicious intent.
